% Abstract
\begin{center}
{\Large{\bf{ABSTRACT}}}
\end{center}

The rise in financial fraud has prompted organizations to collaborate by comparing transaction data across entities. However, this raises critical concerns about user privacy. In order to address these concerns, this work proposes a privacy-preserving fraud detection framework that integrates Laconic Cryptography and Private Set Intersection (PSI) to enable the identification of fraudulent records without disclosing sensitive information.

Our approach uses Laconic Encryption, a cryptographic primitive enabling sublinear communication in just two rounds, based on the Learning With Errors (LWE) assumption. One party generates a succinct digest of encrypted data, which the other uses to perform secure computations\textemdash reducing communication and computation overhead, and ensuring scalability.

Unlike traditional fraud detection systems with centralized databases or high-cost multi-party computation, our framework integrates Laconic PSI tailored for financial scenarios. It securely computes intersections between encrypted datasets, such as transaction logs and fraud records, without exposing unmatched data. By leveraging Merkle Trees, SIS, and dual-Regev encryption, the system ensures strong privacy, quantum resistance, and practical efficiency.

This framework bridges the gap between privacy, performance, and real-world deployment, enabling secure cross-border transaction monitoring, fraud detection across institutions, and insurance claim verification\textemdash while complying with regulations like GDPR and preserving user trust.
